## First Comment (post immediately after submission)

Hi HN,

I'm building AI4Change — an open-source initiative that connects communities facing real-world problems with global contributors who use AI as their tool to build solutions.

**The inspiration**: Playing for Change (https://playingforchange.com) films musicians on street corners across the globe, each playing their part of the same song independently. Layered together, the result is something no single studio could produce. We apply the same model to AI-powered solutions.

**How it works**:

1. Anyone submits a real-world problem (via GitHub Issues)
2. Community discusses and signals which problems matter most
3. Top problems become pilot projects ("songs")
4. Contributors worldwide collaborate — each using AI tools as their instrument
5. Everything ships open source (MIT)

**What makes this different from $EXISTING_THING**:

- vs. hackathons: We build continuously, not for 48 hours
- vs. nonprofits: You need a GitHub account, not a funding cycle
- vs. AI ethics orgs: We ship code, not papers
- vs. typical OSS: We start with the problem, not the solution

**Problem areas we're collecting**: agriculture (crop disease prediction), education (offline literacy tools), health (disease outbreak tracking), environment (air quality monitoring), disaster response (early warning systems).

**Current state**: Phase 1 — collecting seed problems, building the community foundation. No code yet for solutions, which is intentional. The community chooses what we build. The tech stack for each project will be driven by the problem requirements.

**Technical details**: GitHub-native workflow. Problems are Issues, community discussion lives in GitHub Discussions, the website is static on GitHub Pages. Each pilot project gets its own repo. We're particularly interested in edge deployment, offline capability, and low-resource hardware — because the target users often don't have reliable internet or expensive devices.
