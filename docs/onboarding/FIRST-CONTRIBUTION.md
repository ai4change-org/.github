# Your First Contribution

A step-by-step guide to making your first contribution to AI4Change — whether you're a developer, a community member, a designer, or anyone who wants to help.

---

## Before You Start

You'll need:

- [ ] A **GitHub account** (free — [sign up here](https://github.com/signup))
- [ ] 15 minutes of time
- [ ] A problem, a skill, or just curiosity

That's it. No special tools, no development environment, no prior experience required.

---

## Choose Your Path

AI4Change welcomes many kinds of contributions. Pick the one that fits you best:

| Path | Best For | Time Needed | Technical Level |
|:-----|:---------|:------------|:---------------|
| [Submit a Problem](#path-1-submit-a-problem) | Community members with lived experience | 15 min | None |
| [Join a Discussion](#path-2-join-a-discussion) | Anyone with an opinion or expertise | 10 min | None |
| [Improve Documentation](#path-3-improve-documentation) | Writers, editors, translators | 30 min | Basic Git |
| [Contribute Code](#path-4-contribute-code) | Developers, data scientists | 1+ hours | Intermediate |
| [Spread the Word](#path-5-spread-the-word) | Anyone who believes in the mission | 5 min | None |

## Path 1: Submit a Problem

**The most important contribution.** AI4Change starts with problems — real challenges faced by real communities.

### Steps

1. **Go to the Problems repo**: [github.com/ai4change-org/problems/issues/new](https://github.com/ai4change-org/problems/issues/new)

2. **Fill out the issue template** with:
   - **What's the problem?** — Describe the challenge in specific terms
   - **Who is affected?** — Which community, how many people, where
   - **What's been tried?** — Any existing approaches and why they fall short
   - **How could AI help?** — Even a rough idea is valuable
   - **Your connection** — How do you know about this problem?

3. **Submit the issue**

4. **Stay engaged** — respond to questions and comments from the community

### Tips for a Great Problem Submission

- **Be specific**: "Farmers in my district lose 30% of crops to late blight detected too late" beats "Agriculture needs AI"
- **Ground it in experience**: The best problems come from people who've seen them firsthand
- **Include context**: Location, scale, available infrastructure (smartphones? internet? electricity?)
- **Don't worry about solutions**: We'll figure that out together — your job is to describe the problem clearly

### Example

> **Problem**: Maternal health workers in rural clinics in [region] track patient visits and risk factors on paper. High-risk pregnancies are often identified too late because patterns across patients aren't visible in paper records.
>
> **Who's affected**: ~2,000 pregnant women per year across 12 clinics served by 30 community health workers.
>
> **What's been tried**: Paper-based checklists, monthly reports sent to district office. Data arrives too late to act on.
>
> **How AI could help**: A simple mobile app that flags high-risk patterns as health workers enter basic visit data. Doesn't need to be fancy — just needs to work offline on cheap Android phones.

## Path 2: Join a Discussion

Community discussions shape what AI4Change builds and how.

### Steps

1. **Go to Discussions**: [github.com/orgs/ai4change-org/discussions](https://github.com/orgs/ai4change-org/discussions)

2. **Browse existing threads** — find one that interests you

3. **Add your perspective**:
   - Respond to a problem submission with domain expertise
   - Suggest approaches to a challenge
   - Ask clarifying questions
   - Share relevant research or prior art

4. **Or start a new discussion**:
   - Propose an idea with the `idea` tag
   - Ask a question with the `question` tag
   - Share something you've built with `show-and-tell`

### Discussion Norms

- Lead with curiosity: "Tell me more about that" beats "That won't work"
- Be specific: Back up opinions with reasoning or evidence
- Respect time zones: Async-first — don't expect immediate responses
- Credit sources: Link to research, data, or prior art

## Path 3: Improve Documentation

Good documentation removes barriers. If you can read and write, you can improve docs.

### Steps

1. **Fork the repository** you want to improve:
   - Organization docs: [github.com/ai4change-org/.github](https://github.com/ai4change-org/.github)
   - Website: [github.com/ai4change-org/ai4change-org.github.io](https://github.com/ai4change-org/ai4change-org.github.io)

2. **Find something to improve**:
   - Fix a typo or unclear sentence
   - Add a missing explanation
   - Translate content into your language
   - Improve formatting or readability

3. **Make your changes** on a new branch

4. **Submit a Pull Request** with:
   - A clear title describing the change
   - A brief explanation of why the change improves things

5. **Respond to review feedback** — maintainers may suggest tweaks

### Quick Edit (No Git Required)

For small changes, you can edit directly on GitHub:
1. Navigate to the file you want to change
2. Click the pencil icon (edit button)
3. Make your changes
4. Click "Propose changes" — GitHub creates the fork and PR for you

## Path 4: Contribute Code

For developers ready to build. Code contributions will become more active once pilot projects launch (Phase 3).

### Steps

1. **Find an issue** labeled `good first issue`:
   - Check across all [AI4Change repos](https://github.com/ai4change-org)
   - Look for issues with clear descriptions and acceptance criteria

2. **Comment on the issue** to signal you're working on it

3. **Fork and clone** the repository

4. **Create a branch** with a descriptive name:
   ```bash
   git checkout -b fix/improve-problem-template
   ```

5. **Make your changes**, following:
   - The project's existing code style
   - Conventional Commits for commit messages (`feat:`, `fix:`, `docs:`)
   - The project's README for setup instructions

6. **Test your changes** locally

7. **Submit a Pull Request** with:
   - A clear title and description
   - Reference to the issue it addresses (`Closes #123`)
   - Screenshots or demos if applicable

8. **Respond to code review** — maintainers will review and may request changes

### Code Contribution Checklist

- [ ] Changes address a specific issue or discussion
- [ ] Code follows the project's existing style
- [ ] Changes are documented where needed
- [ ] Commit messages follow Conventional Commits
- [ ] PR description explains what and why

## Path 5: Spread the Word

Sharing AI4Change with the right people is a genuine contribution.

### Steps

1. **Star the GitHub org**: [github.com/ai4change-org](https://github.com/ai4change-org) — stars help visibility

2. **Share on social media**:
   - Twitter/X: Tag [@ai4changeorg](https://twitter.com/ai4changeorg)
   - LinkedIn: Share with your professional network
   - Use `#AI4Change` and `#OpenSource`

3. **Tell someone specific**:
   - A colleague who works in social impact
   - A community leader who faces a real problem
   - A developer looking for meaningful open-source work
   - A teacher, farmer, or health worker who could submit a problem

4. **Write about it**: Blog posts, newsletter mentions, community forums
