# 🔒 Security Policy

## Supported Versions

| Repository | Supported |
|:-----------|:---------:|
| [ai4change-org.github.io](https://github.com/ai4change-org/ai4change-org.github.io) | ✅ |
| [problems](https://github.com/ai4change-org/problems) | ✅ |
| [.github](https://github.com/ai4change-org/.github) | ✅ |

All active repositories in the AI4Change organization are covered by this policy.

## Reporting a Vulnerability

We take the security of AI4Change and its community seriously. If you discover a security vulnerability, we appreciate your help in disclosing it responsibly.

### How to Report

1. **DO NOT** open a public GitHub issue for security vulnerabilities
2. Email us at **security@ai4change.org** (or open a [private security advisory](https://github.com/ai4change-org/.github/security/advisories/new))
3. Include the following in your report:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

| Timeline | Action |
|:---------|:-------|
| **24 hours** | We'll acknowledge receipt of your report |
| **72 hours** | We'll provide an initial assessment |
| **7 days** | We'll have a remediation plan |
| **30 days** | We aim to resolve the issue |

### Our Commitments

- We will **not** take legal action against researchers who follow this policy
- We will work with you to understand and resolve the issue
- We will credit you in the fix (unless you prefer anonymity)
- We will notify affected users if necessary

### Scope

This policy applies to:
- All code in AI4Change repositories
- The AI4Change website (ai4change-org.github.io)
- Any infrastructure or services operated by AI4Change

### Out of Scope

- Social engineering attacks against AI4Change contributors
- Denial of service attacks
- Issues in third-party dependencies (please report to the upstream project)

## Security Best Practices for Contributors

When contributing to AI4Change projects:

- **Never** commit secrets, API keys, or credentials
- Use environment variables for sensitive configuration
- Keep dependencies up to date
- Follow the principle of least privilege
- Report suspicious activity immediately
